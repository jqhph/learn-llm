二、Prometheus 指标类型

类型语义RAG 示例PromQL 模式
Counter只增不减请求总数、错误总数、token 总数rate(rag_requests_total[5m])
Gauge可增可减文档数、活跃连接数、检索平均分rag_retrieval_score
Histogram分布统计延迟分布、检索 chunk 数分布histogram_quantile(0.95, rate(...))

三、核心 PromQL 查询

查询目标PromQL
QPS（每秒请求数）rate(rag_requests_total{endpoint="/ask"}[5m])
P95 延迟（毫秒）histogram_quantile(0.95, rate(rag_request_duration_ms_bucket{endpoint="/ask"}[5m]))
错误率sum(rate(rag_requests_total{status=~"5.."}[5m])) / sum(rate(rag_requests_total[5m]))
Token 消耗速率rate(rag_tokens_total{token_type="total"}[1h])
服务器 CPU 使用率100 - avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[2m])) * 100
磁盘剩余比例node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"}
容器内存用量container_memory_usage_bytes{container_label_com_docker_compose_service="rag-api"}

五、FastAPI 指标埋点代码片段

📦 metrics.py — 指标定义

from prometheus_client import Counter, Histogram, Gauge

rag_requests_total = Counter(
    "rag_requests_total", "总请求数",
    labelnames=["endpoint", "method", "status"])

rag_request_duration_ms = Histogram(
    "rag_request_duration_ms", "延迟(ms)",
    labelnames=["endpoint"],
    buckets=[50,100,200,500,1000,2000,5000,10000,30000,60000])

rag_tokens_total = Counter(
    "rag_tokens_total", "token 消耗",
    labelnames=["model", "token_type"])

🔌 main.py — /metrics 端点 + 中间件

from .metrics import metrics_middleware, get_metrics

# 注册中间件
app.middleware("http")(metrics_middleware)

# 注册 metrics 端点
@app.get("/metrics")
async def metrics():
    from fastapi.responses import PlainTextResponse
    return PlainTextResponse(content=get_metrics())
