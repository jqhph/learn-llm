四个核心指标速览

指标评估对象回答的问题评分方式阈值参考
Faithfulness 答案 vs 上下文 答案有无"看到"上下文？ 拆 claims → 逐条验证 → 比例 >0.9 优秀 / <0.7 严重
Answer Relevancy 答案 vs 用户查询 答案是否在答所问？ LLM 直接打分 (0-5) 或反向生成问题算相似度 >4.0 优秀 / <3.0 需改进
Context Precision 上下文 vs 用户查询 检索的 chunk 有多少相关？ 逐条判断相关性 → 加权精度（排序敏感） >0.8 优秀 / <0.6 噪音太多
Context Recall 上下文 vs 生成答案 回答所需信息是否都在？ 拆 claims → 逐条匹配上下文 >0.8 优秀 / <0.6 缺信息

指标分两组

组别指标诊断方向
检索侧Context Precision + Context Recall检索系统是否找到了对的内容？
生成侧Faithfulness + Answer RelevancyLLM 是否正确使用了检索到的内容？

⚠️ 如果 Faithfulness 低但 Context Recall 高 → 问题在生成侧（prompt/模型）。反之亦然。

LLM-as-a-Judge 调用模板

from openai import OpenAI
client = OpenAI()

def _judge(system, user, model="gpt-4o-mini"):
    resp = client.chat.completions.create(
        model=model, temperature=0.0,
        messages=[
            {"role": "system", "content": system},
            {"role": "user", "content": user},
        ],
    )
    return resp.choices[0].message.content

生产环境策略

  • 采样评估：1-5% 用户请求异步跑 LLM-as-a-Judge
  • 退化告警：滑动窗口 Faithfulness < 基线 → 告警

  • 人工抽检：每周从低分样本抽 10 条验证
