核心评估指标速算

指标公式取值区间解读
Recall@K |retrieved[:K] ∩ relevant| / |relevant| [0, 1] K 个结果里覆盖了多少正确答案。目标 > 0.85
MRR mean(1/rank_of_first_relevant) [0, 1] 第一个正确答案排第几。目标 > 0.75
NDCG DCG / IDCG (分级相关度的累计增益折现比) [0, 1] 排序质量（考虑分级相关度和排名位置）。目标 > 0.80

recall_at_k
def recall_at_k(retrieved_ids, relevant_ids, k):
    if not relevant_ids:
        return 0.0
    hits = len(set(retrieved_ids[:k]) & set(relevant_ids))
    return hits / len(relevant_ids)

mrr
def mrr(retrieved_ids, relevant_ids):
    for i, rid in enumerate(retrieved_ids):
        if rid in relevant_ids:
            return 1.0 / (i + 1)
    return 0.0

重排序器代码骨架

pip install sentence-transformers
from sentence_transformers import CrossEncoder

model = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")
pairs = [["query", doc["text"]] for doc in documents]
scores = model.predict(pairs, show_progress_bar=False)

for i, d in enumerate(documents):
    d["rerank_score"] = float(scores[i])

ranked = sorted(documents, key=lambda d: d["rerank_score"], reverse=True)
top_n = ranked[:3]

检索失败模式诊断

同义词鸿沟

"退款" vs "退费"、"发货" vs "配送"——embedding 没区分

对策：模型升级、同义词扩展

跨 chunk 答案

答案分散在多个 chunk 中

对策：增大 chunk_size / overlap、提高 top-K

关键词覆盖不足

生僻词/缩写/SKU 号没命中

对策：混合检索（向量+BM25）

粒度错配

概括性问题 vs 细粒度 chunk

对策：分层索引、LLM 聚合
